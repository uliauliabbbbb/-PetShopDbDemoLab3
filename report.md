**Тема:** подключение к базам данных через ODBC, OleDb и LINQ.

**Среда:** Visual Studio / .NET 8, PostgreSQL 15+, Windows 11.

## Цель работы

Научиться подключаться к базе данных тремя разными способами: через ODBC, OleDb и LINQ (EF Core). По заданию подключения должны быть реализованы в одном приложении, при этом разные блоки могут работать как с одной, так и с разными базами. Результат выборки нужно вывести в стандартный элемент (допускается консоль).

## Задание

1. Подключиться к БД через ODBC и сделать выборку.
2. Подключиться к БД через OleDb и сделать выборку.
3. Подключиться к БД через LINQ и сделать выборку.
4. Объединить всё в одном приложении и оформить вывод.

## Описание предметной области

В качестве учебной БД я выбрала зоомагазин (интернет-магазин товаров для животных). Эту же базу я планирую использовать дальше в проекте по веб-программированию (ASP.NET Core Web API + React), поэтому решила сразу собрать её нормально, с категориями, корзинами и заказами.

Структура БД (PostgreSQL, база `petshop`):

| Таблица      | Назначение                                   |
|--------------|----------------------------------------------|
| categories   | категории товаров (Корм, Игрушки и т.д.)     |
| products     | сами товары, ссылается на categories         |
| users        | покупатели                                   |
| carts        | корзина каждого пользователя                 |
| cart_items   | позиции в корзине                            |
| orders       | оформленные заказы                           |
| order_items  | позиции заказа                               |

Тестовые данные: 20 товаров, 3 пользователя, 3 заказа. Скрипт создания базы и наполнения лежит в файле `petshop_schema.sql`.

## Используемые технологии

- .NET 8, C#
- `System.Data.Odbc` (блок ODBC)
- `System.Data.OleDb` (блок OleDb)
- `Microsoft.EntityFrameworkCore` + `Npgsql.EntityFrameworkCore.PostgreSQL` (блок LINQ)
- PostgreSQL ODBC Driver (psqlODBC), драйвер для ODBC
- Microsoft.ACE.OLEDB.16.0, провайдер для OleDb

## Структура программы

Программа консольная, состоит из трёх классов-модулей:

- `OdbcDemo`: подключается к PostgreSQL через ODBC и выводит список товаров.
- `OleDbDemo`: подключается через OleDb-провайдер и выводит список заказов.
- `LinqDemo`: через EF Core с LINQ считает статистику по пользователям.

Точка входа `Main` последовательно вызывает все три блока и в конце собирает HTML-отчёт `report.html`, который автоматически открывается в браузере. HTML-отчёт сделала дополнительно, чтобы было нагляднее, чем просто текст в консоли.

*Скриншот 2: результат запроса `SELECT * FROM products LIMIT 10;` в pgAdmin.*

### 2. Блок ODBC

Для ODBC поставила PostgreSQL ODBC Driver (psqlODBC) с официального сайта. После установки в «ODBC Data Sources (64-bit)» появился драйвер «PostgreSQL Unicode(x64)».

В коде подключение собирается через строку соединения с указанием драйвера, сервера, порта, базы и учётной записи:

```csharp
"Driver={PostgreSQL Unicode(x64)};Server=localhost;Port=5432;Database=petshop;Uid=postgres;Pwd=1234;"
```

Через `OdbcCommand` отправляю SQL-запрос с JOIN таблиц `products` и `categories`, читаю результат через `OdbcDataReader` и складываю строки в список `List<ProductRow>` для последующего вывода (и в консоль, и в HTML).

*Скриншот 3: окно «ODBC Data Sources (64-bit)» со списком драйверов, где видно psqlODBC.*
*Скриншот 4: консольный вывод блока 1 (ODBC), таблица с товарами.*

### 3. Блок OleDb

Здесь была сложность. В современном .NET `System.Data.OleDb` больше не поддерживает мост MSDASQL поверх ODBC, а нативного OleDb-провайдера для PostgreSQL не существует. Поэтому подключиться через OleDb напрямую к Postgres у меня не получилось.

В задании сказано, что разные блоки могут работать с разными источниками, поэтому я сделала так: сначала программа выгружает заказы из PostgreSQL в файл `data/orders.csv` (через уже знакомое ODBC-соединение), а затем подключается к этому CSV провайдером `Microsoft.ACE.OLEDB.16.0` с Text-драйвером. Чтобы провайдер правильно распарсил CSV, программа автоматически генерирует рядом файл `schema.ini` с описанием колонок и кодировки.

Строка подключения для OleDb выглядит так:

```csharp
"Provider=Microsoft.ACE.OLEDB.16.0;Data Source=<папка с csv>;Extended Properties=\"text;HDR=Yes;FMT=Delimited\";"
```

Далее идёт обычная схема: `OleDbCommand` + `OleDbDataReader`, и построчное чтение.

*Скриншот 5: консольный вывод блока 2 (OleDb), таблица с заказами и сообщением про CSV.*

### 4. Блок LINQ

Для LINQ драйверы устанавливать не нужно, всё ставится через NuGet (`Npgsql.EntityFrameworkCore.PostgreSQL`). Описала классы-сущности `User` и `Order`, отношение «один ко многим», а в `DbContext` через Fluent API задала маппинг на реальные имена таблиц и колонок (в коде они в `snake_case`, как в PostgreSQL).

LINQ-запрос для статистики:

```csharp
db.Users.Select(u => new {
    u.Name, u.Email,
    OrdersCount = u.Orders.Count(),
    TotalSpent = u.Orders.Sum(o => (decimal?)o.Total) ?? 0m
}).OrderByDescending(x => x.TotalSpent).ToList();
```

Дополнительно вытаскиваю 3 последних заказа с подгрузкой пользователя через `Include`.

*Скриншот 6: консольный вывод блока 3 (LINQ), статистика по пользователям и последние заказы.*

Страница свёрстана таблицами, у каждой секции своя цветовая отметка: ODBC синий, OleDb оранжевый, LINQ бирюзовый. Эта часть к заданию не относится, но делает результат красивее и удобнее для проверки.

## Вывод

В ходе работы я научилась подключаться к PostgreSQL тремя разными способами: ODBC, OleDb и LINQ через EF Core. Каждый из них имеет свои особенности.

ODBC оказался самым «низкоуровневым»: нужно отдельно ставить драйвер, писать сырой SQL, разбирать результат через `DataReader`. Зато он работает почти с любой СУБД, у которой есть ODBC-драйвер.

OleDb по идее похож на ODBC, но в современном .NET его сильно урезали: моста к ODBC больше нет, а провайдеров под Postgres нет в принципе. На практике через OleDb сегодня удобно работать только с Access, Excel и текстовыми файлами через ACE-провайдер, поэтому в этой работе я подключилась к CSV-файлу.

LINQ через EF Core оказался самым удобным: типизированные сущности, маппинг на таблицы, никаких сырых SQL-запросов, всё пишется на C# и проверяется компилятором. Минус в том, что нужно настраивать `DbContext` и маппинг колонок, что для маленького приложения избыточно, но для нормального проекта это себя окупает.

Цель работы достигнута, все три способа подключения реализованы в одном приложении, выборка данных из БД выполнена, результат выведен в консоль и продублирован в HTML-отчёт.
